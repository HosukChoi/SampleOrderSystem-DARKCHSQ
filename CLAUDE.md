# CLAUDE.md — 반도체 시료 생산주문관리 시스템

## 프로젝트 개요

가상의 반도체 회사 **S-Semi**의 시료(Sample) 생산주문관리 콘솔 애플리케이션이다.
연구소·팹리스·대학 연구실 등 고객에게 납품되는 반도체 시료의 주문, 생산, 재고, 출고를 통합 관리한다.

---

## 도메인 용어

| 용어 | 설명 |
|---|---|
| 시료(Sample) | 시스템이 관리하는 기본 단위. 고유 ID, 이름, 평균 생산시간, 수율을 가짐 |
| 수율(Yield) | 정상 시료 수 / 총 생산 시료 수 (예: 90/100 = 0.9) |
| 생산 라인 | 시료를 한 번에 하나씩 생산하는 설비 흐름 |
| 생산 큐 | 생산 대기 중인 작업 목록. FIFO 스케줄링 |

---

## 주문 상태 흐름

```
RESERVED → (승인, 재고 충분)  → CONFIRMED → RELEASE
         → (승인, 재고 부족)  → PRODUCING → CONFIRMED → RELEASE
         → (거절)             → REJECTED
```

| 상태 | 의미 |
|---|---|
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 (모니터링 제외) |
| PRODUCING | 승인 완료, 재고 부족으로 생산 중 |
| CONFIRMED | 승인 완료, 출고 대기 |
| RELEASE | 출고 완료 |

---

## 핵심 기능 명세

### 1. 시료 관리
- **시료 등록**: ID, 이름, 평균 생산시간, 수율 입력
- **시료 조회**: 전체 시료 목록 + 현재 재고 수량 표시
- **시료 검색**: 이름 등 속성으로 검색

### 2. 시료 주문 (예약)
- 입력값: 시료 ID, 고객명, 주문 수량
- 주문 생성 즉시 상태 = `RESERVED`

### 3. 주문 승인 / 거절
- `RESERVED` 목록을 조회 후 처리
- **승인**:
  - 재고 충분 → 즉시 `CONFIRMED`
  - 재고 부족 → 생산 라인에 자동 등록, `PRODUCING`
- **거절**: 즉시 `REJECTED`

### 4. 모니터링
- 상태별 주문 수 조회 (`RESERVED` / `PRODUCING` / `CONFIRMED` / `RELEASE`)
- 시료별 재고 현황 + 재고 상태 표기:
  - **여유**: 주문 대비 재고 충분
  - **부족**: 주문 대비 재고 수량 부족
  - **고갈**: 재고 수량 = 0

### 5. 생산 라인
- **실 생산량**: `ceil(부족분 / (수율 × 0.9))`
- **총 생산시간**: `평균 생산시간 × 실 생산량`
- 생산 완료 시 주문 상태 `PRODUCING` → `CONFIRMED`
- 생산 큐 스케줄링: **FIFO**
- 화면 표시: 현재 생산 중인 시료 정보 + 대기 큐 목록

### 6. 출고 처리
- `CONFIRMED` 상태 주문에 대해 출고 실행
- 출고 후 상태 = `RELEASE`

## 구현 지침

- **실행 방식**: 콘솔(터미널) 기반. GUI 없음.
- **언어/프레임워크**: 별도 지정 없음 — 팀/개인 결정에 따름.
- **UI 레이아웃**: 예시 화면은 참고용이며, 실제 구성은 자유롭게 결정.
- **데이터 영속성**: 과제 명세에 DB/파일 저장 요건 없음 — 인메모리로 구현해도 무방.
- **생산 라인 동시성**: 한 번에 하나의 시료만 생산. 큐는 FIFO.
- **REJECTED 주문**: 유효한 주문이 아니므로 모니터링·통계에서 제외.

## 재고 관리 정책

### 재고 개념 구분

| 구분 | 정의 |
|---|---|
| **실재 재고** | 현재 실제로 보유 중인 시료 수량. 생산 완료 시마다 즉시 반영되며 생산 중에도 실시간 업데이트 |
| **필요 재고** | 아직 출고되지 않은 승인 주문(`CONFIRMED` + `PRODUCING`)이 요구하는 총 수량 |
| **가용 재고** | `실재 재고 - 필요 재고` — 새 주문을 즉시 처리할 수 있는 여유분 |

### 주문 승인 시 상태 결정 로직

```
가용 재고 = 실재 재고 - 필요 재고

가용 재고 >= 주문 수량  →  CONFIRMED  (즉시 출고 가능)
가용 재고 <  주문 수량  →  PRODUCING  (부족분 생산 필요)
```

- 필요 재고와 실재 재고를 **모두** 고려하여 결정한다.
- 실재 재고가 있더라도 다른 주문에 이미 예약된 재고가 있으면 가용 재고가 부족할 수 있다.

### PRODUCING 불변 원칙

- 주문이 한 번 `PRODUCING` 상태로 전환되면 **무조건 생산을 완료**한다.
- 중간에 재고가 충분해지더라도 생산을 취소하거나 건너뛰지 않는다.
- 생산 완료 후 실재 재고에 반영되고, 주문 상태가 `CONFIRMED`로 전환된다.

### 모니터링 재고 표시 규칙

- **표시 수치**: 실재 재고 (실제 보유량)
- **상태 판단**: 가용 재고(`실재 재고 - 필요 재고`) 기준

| 상태 | 조건 |
|---|---|
| **여유** | 가용 재고 > 0 |
| **부족** | 가용 재고 ≤ 0 (단, 실재 재고 > 0) |
| **고갈** | 실재 재고 = 0 |

### 실시간 재고 업데이트

- 생산 라인에서 시료가 한 개 완성될 때마다 실재 재고를 즉시 증가시킨다.
- 출고(`RELEASE`) 처리 시 실재 재고에서 주문 수량을 차감한다.
- 필요 재고는 주문 상태 변경(`CONFIRMED`/`PRODUCING` 추가·제거) 시마다 재계산한다.
