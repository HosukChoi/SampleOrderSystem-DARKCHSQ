**Product Name:** S-Semi Sample Order Management System  
**Version:** 1.0  
**Author:** hsw.choi  
**Date:** 2026-06-12

---

## 1. 개요

### 1.1 배경

반도체 회사 **S-Semi**는 연구소·팹리스·대학 연구실 등 외부 고객에게 반도체 시료(Sample)를 소량 제작·납품한다. 기존에는 주문 접수부터 출고까지의 흐름을 수작업으로 관리해 왔으며, 재고 파악 오류·생산 일정 충돌·출고 누락 등의 문제가 반복되었다.

### 1.2 목적

시료의 주문 접수 → 생산 → 재고 관리 → 출고까지의 전 과정을 하나의 콘솔 애플리케이션으로 통합하여, 담당자가 실시간으로 현황을 파악하고 신속하게 처리할 수 있도록 한다.

### 1.3 범위

- 시료 등록 및 조회
- 고객 주문 접수 / 승인 / 거절
- 생산 라인 스케줄링 및 실시간 진행 현황
- 재고 실시간 추적
- 출고 처리

## 3. 도메인 용어 정의

| 용어 | 정의 |
|---|---|
| 시료 (Sample) | 시스템이 관리하는 생산·주문 단위. ID, 이름, 평균 생산시간(초/개), 수율을 속성으로 가진다 |
| 수율 (Yield) | 정상품 수 / 총 생산 수. 예: 0.9 = 90% |
| 실재 재고 | 현재 실제로 보유 중인 수량. 생산 완료 시마다 즉시 반영 |
| 필요 재고 | CONFIRMED + PRODUCING 상태 주문이 요구하는 총 수량 |
| 가용 재고 | 실재 재고 − 필요 재고. 신규 주문을 즉시 처리할 수 있는 여유분 |
| 생산 라인 | 시료를 한 번에 하나씩 생산하는 단일 설비 흐름 |
| 생산 큐 | 생산 대기 중인 작업 목록. FIFO 순서로 처리 |

## 4. 주문 상태 흐름

```
주문 접수
   │
RESERVED ──(승인, 가용재고 충분)──► CONFIRMED ──► RELEASE
   │                                                  ▲
   ├──(승인, 가용재고 부족)──► PRODUCING ─────────────┘
   │
   └──(거절)──► REJECTED  (이후 모든 통계·모니터링에서 제외)
```

| 상태 | 의미 |
|---|---|
| RESERVED | 주문 접수 완료, 처리 대기 중 |
| CONFIRMED | 승인 완료, 출고 대기 |
| PRODUCING | 재고 부족으로 생산 진행 중 |
| RELEASE | 출고 완료 |
| REJECTED | 주문 거절 |

## 5. 핵심 기능 요구사항

### 5.1 시료 관리

| ID | 요구사항 |
|---|---|
| F-SAM-01 | 시료를 등록할 수 있다. 입력 항목: ID (정수), 이름, 평균 생산시간(초), 수율(%) |
| F-SAM-02 | 전체 시료 목록을 조회할 수 있다. 각 시료의 현재 실재 재고와 재고 상태(여유/부족/고갈)를 함께 표시한다 |
| F-SAM-03 | 시료를 ID 또는 이름 키워드로 검색할 수 있다 |
| F-SAM-04 | 이미 존재하는 ID로 등록을 시도하면 오류를 안내하고 등록을 거부한다 |

### 5.2 시료 주문

| ID | 요구사항 |
|---|---|
| F-ORD-01 | 고객이 특정 시료를 수량 지정하여 주문할 수 있다. 입력 항목: 시료 ID, 고객명, 주문 수량 |
| F-ORD-02 | 주문 생성 즉시 상태는 RESERVED가 된다 |
| F-ORD-03 | 존재하지 않는 시료 ID로 주문하면 오류를 안내한다 |

### 5.3 주문 승인 / 거절

| ID | 요구사항 |
|---|---|
| F-APR-01 | RESERVED 상태의 주문 목록을 조회할 수 있다 |
| F-APR-02 | 주문을 승인할 수 있다. 가용 재고 ≥ 주문 수량이면 CONFIRMED로 즉시 전환된다 |
| F-APR-03 | 주문을 승인할 때 가용 재고 < 주문 수량이면 PRODUCING으로 전환하고 생산 큐에 자동 등록한다 |
| F-APR-04 | 주문을 거절할 수 있다. 거절된 주문은 REJECTED 상태가 되며 이후 통계에서 제외된다 |
| F-APR-05 | 가용 재고 계산 시 실재 재고와 이미 다른 주문에 예약된 필요 재고를 모두 반영한다 |

### 5.4 모니터링

| ID | 요구사항 |
|---|---|
| F-MON-01 | 상태별(RESERVED / PRODUCING / CONFIRMED / RELEASE) 주문 건수를 조회할 수 있다 |
| F-MON-02 | 주문량 확인 화면에서 현재 생산 큐를 FIFO 순으로 상세 조회할 수 있다. 표시 항목: 순서, 주문ID, 시료 이름, 주문량, 부족분, 실생산량, 예상완료시각(HH:MM:SS) |
| F-MON-03 | 시료별 재고 현황을 조회할 수 있다. 표시 항목: 실재 재고, 필요 재고, 가용 재고, 재고 상태 |
| F-MON-04 | 재고 상태는 가용 재고 기준으로 판단한다: 여유(>0) / 부족(≤0, 단 실재>0) / 고갈(실재=0) |

### 5.5 생산라인 조회

| ID | 요구사항 |
|---|---|
| F-PRD-01 | 현재 생산 중인 작업의 진행 상황(시료 이름, 생산 수량 / 목표 수량)을 조회할 수 있다 |
| F-PRD-02 | 대기 중인 작업 목록을 FIFO 순으로 상세 조회할 수 있다. 표시 항목: 순서, 주문ID, 시료 이름, 주문량, 부족분, 실생산량, 예상완료시각(HH:MM:SS) |
| F-PRD-03 | 실 생산량은 `ceil(부족분 / (수율 × 0.9))`로 계산한다 |
| F-PRD-04 | 생산 큐는 FIFO로 스케줄링되며, 한 번에 하나의 작업만 진행된다 |
| F-PRD-05 | PRODUCING 상태에 진입한 작업은 중간에 재고가 충분해지더라도 반드시 완료한다 |
| F-PRD-06 | 생산이 완료되면 실재 재고에 즉시 반영되고 해당 주문은 CONFIRMED로 전환된다 |

### 5.6 출고 처리

| ID | 요구사항 |
|---|---|
| F-SHP-01 | CONFIRMED 상태의 주문 목록을 조회할 수 있다. 표시 항목: No, 주문ID, 고객명, 시료 이름, 수량 |
| F-SHP-02 | 주문 ID를 지정하여 출고 처리할 수 있다. 처리 후 상태는 RELEASE가 된다 |
| F-SHP-03 | 출고 처리 시 실재 재고에서 주문 수량을 즉시 차감한다 |
| F-SHP-04 | CONFIRMED 상태가 아닌 주문에 대한 출고 시도는 실패 처리한다 |

|---|---|
| NF-UI-01 | 모든 테이블의 헤더와 데이터 행은 한글 이중폭 문자를 고려한 display-width 기준으로 정렬된다 |

### 8.2 재고 계산 흐름

```
승인 요청
  │
  ├─ 가용 재고 = 실재 재고 − 필요 재고
  │
  ├─ 가용 재고 >= 주문 수량  ──► CONFIRMED
  │
  └─ 가용 재고 <  주문 수량  ──► PRODUCING
       실 생산량 = ceil(부족분 / (수율 × 0.9))
       생산 큐 enqueue
```

## 10. 제약 사항 및 가정

- GUI 없음. 콘솔(터미널) 전용 애플리케이션이다.
- 생산 라인은 단일 라인으로 동시에 하나의 작업만 처리한다.
- 멀티스레딩 없이 단일 스레드로 동작하며, `tick()` 호출 시점에 생산 진행을 갱신한다.
- 네트워크 기능 없음. 로컬 단독 실행 환경을 가정한다.
- REJECTED 주문은 통계·모니터링 집계에서 완전히 제외한다.
